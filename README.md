# Provenance Guard

A backend API for detecting whether submitted creative writing was written by a human or generated by AI. Built for AI201 Project 4.

## Architecture Overview

A submitted piece of text enters at POST /submit. It passes through two independent detection signals: an LLM classifier (Groq) and a stylometric heuristics function. Each returns a score between 0.0 and 1.0. A confidence scorer combines them into a single weighted score. That score maps to one of three transparency labels returned to the caller. Every submission is written to a structured audit log. Appeals enter at POST /appeal, update the original entry status to "under_review", and append the creator's reasoning.

## Detection Signals

### Signal 1: LLM Classification (Groq llama-3.3-70b-versatile)
Sends the text to Groq with a prompt asking it to assess whether the content reads as human or AI-generated. Returns a float between 0.0 (definitely human) and 1.0 (definitely AI). Captures semantic and stylistic coherence holistically. Blind spot: can flag formal or clean human writing as AI, and can be fooled by lightly edited AI output.

### Signal 2: Stylometric Heuristics
Computes three structural properties of the text: sentence length variance, type-token ratio (vocabulary diversity), and punctuation density. AI text tends to be more uniform; human writing is more variable. Returns a float between 0.0 and 1.0. Blind spot: short texts lack enough signal, and formal human writing looks structurally similar to AI output.

## Confidence Scoring

Combined score = (llm_score * 0.6) + (stylo_score * 0.4)

The LLM signal is weighted higher because it captures meaning and style, not just structure. The stylometric signal is a useful structural check but less reliable on short or formal text.

Thresholds:
- >= 0.75: Likely AI-generated
- <= 0.40: Likely human-written
- 0.41-0.74: Uncertain

The AI threshold is set at 0.75 rather than 0.5 because false positives (labeling a human's work as AI) are worse than false negatives on a creative platform.

### Example Submissions

High-confidence AI input:
"Artificial intelligence represents a transformative paradigm shift in modern society. It is important to note that while the benefits of AI are numerous, it is equally essential to consider the ethical implications."
- llm_score: 0.92, stylo_score: 0.78, confidence: 0.86
- Label: Likely AI-generated

Lower-confidence input:
"ok so i finally tried that new ramen place downtown and honestly? underwhelming. the broth was fine but they put WAY too much sodium in it."
- llm_score: 0.18, stylo_score: 0.31, confidence: 0.23
- Label: Likely human-written

## Transparency Label Variants

High-confidence AI:
"Likely AI-generated. This content was flagged as probably created by an AI writing tool. Confidence: High."

High-confidence human:
"Likely human-written. This content appears to have been written by a person. Confidence: High."

Uncertain:
"Uncertain. Our system could not confidently determine whether this content was written by a human or AI. The author may appeal if this label is incorrect."

## Rate Limiting

Limit: 10 requests per minute, 100 requests per day per IP address.

Reasoning: a legitimate creator submitting their own work would rarely need more than a few submissions per minute. 10/minute allows normal usage while blocking scripts trying to flood the system. 100/day is generous for a single user while still preventing automated abuse.

## Known Limitations

Stylometric heuristics perform poorly on short texts. A single sentence or a short poem does not provide enough variance data to produce a meaningful signal. The system falls back to a neutral 0.5 score for texts with fewer than 2 sentences, which pushes the result toward the uncertain range regardless of the LLM signal.

## Spec Reflection

The spec helped most during confidence scoring design. Writing out the thresholds in planning.md before coding forced a decision about what 0.6 actually means to a user, which made the label logic straightforward to implement.

One divergence: the planning doc assumed stylometrics would produce more varied scores across inputs. In practice the signal clusters around 0.5 for short texts, which means most short submissions land in the uncertain range. A future version would require a minimum text length before running stylometrics.

## API Endpoints

POST /submit
- Body: { "text": "...", "creator_id": "..." }
- Returns: { content_id, attribution, confidence, label, llm_score, stylo_score }

POST /appeal
- Body: { "content_id": "...", "creator_reasoning": "..." }
- Returns: { message, content_id, status }

GET /log
- Returns: { entries: [...] }
